# Excalidrow

A premium-quality, **local-first**, single-user whiteboard studio inspired by Excalidraw, Linear, Notion and tldraw. Pure JavaScript end-to-end — no TypeScript, no Docker, no cloud services. Everything runs on your machine.

> Sketch, diagram, brainstorm and architect — without shipping a single byte to the cloud.

---

## ✨ Highlights

- **Excalidraw-powered editor** with infinite canvas, freehand, shapes, multi-select, undo/redo, copy-paste, alignment guides and more — wrapped in a custom premium chrome.
- **Multi-page boards** with autosave, version history, and on-the-fly thumbnail capture.
- **Premium dashboard** with sidebar, search, favorites, archived view, folders, grid/list view, command palette (`⌘K`), keyboard shortcuts panel, and Linear-style polish.
- **Local file storage** via Multer — uploads land in `uploads/` on disk; thumbnails are generated by Sharp.
- **Realtime architecture** built on Socket.IO + Yjs (CRDT) so multi-window/multi-user collaboration is one config change away.
- **AI-ready** placeholder routes (`/api/v1/ai/*`) for prompt-to-diagram, text-to-flowchart, UML and mindmap features. Wire your model of choice (Ollama, OpenAI, Claude, …) into one controller.
- **Export to PNG / SVG / PDF / JSON** — all rendered client-side, persisted to disk on the server.
- **Hardened API** — Helmet, CORS, rate-limiting, refresh-token rotation with reuse detection, hpp, request IDs, structured pino logs, soft-delete for boards.
- **Pure JavaScript** — ESM, no build step required for the API; Vite bundles the web app.

### Data model

| Collection      | Purpose                                                                |
|-----------------|------------------------------------------------------------------------|
| `users`         | account + `preferences { theme, defaultFont, snapToGrid, … }`         |
| `refreshtokens` | rotated JTI store with reuse detection + TTL index                    |
| `boards`        | top-level whiteboard, soft-deleted, indexed for search and favorites   |
| `pages`         | scene snapshot (`elements`, `appState`, `files`) + capped version log |
| `folders`       | personal organisation                                                 |
| `assets`        | every uploaded image / export / thumbnail with sha256                  |
| `activities`    | audit log surfaced on the activity page                                |
| `notifications` | local notification stream (export complete, etc.)                     |

### Realtime

`apps/api/src/sockets/index.js` authenticates with the same JWT used for HTTP, joins clients into rooms keyed by `boardId:pageId`, and forwards Yjs document/awareness updates to every other peer. The first client to enter a room rehydrates the doc from `pages.yDoc`; updates are debounced and persisted back to disk. Presence is in-memory (single node) — swap in Redis pub/sub if you ever want multi-node.

### Local storage

Everything lives under `uploads/` at the repo root:

```
uploads/
├── boards/        # board-level snapshots (reserved)
├── images/        # user-uploaded images (originals)
├── exports/       # PNG / SVG / PDF / JSON exports
└── thumbnails/    # 320px webp previews + board cover thumbs
```

The Express app serves them statically under `/uploads/*` with `Cross-Origin-Resource-Policy: cross-origin` so the web app can render them without CORS pain.

## 🚀 Getting started

### Prerequisites

- **Node.js ≥ 18.18**
- **MongoDB ≥ 6** running locally on `mongodb://127.0.0.1:27017`
  - macOS: `brew services start mongodb-community`
  - Windows: install MongoDB Community Server and start the service
  - Linux: `sudo systemctl start mongod`

### 1. Install

```bash
npm install
```

This installs everything for the API, web app and the `@excalidrow/shared` package via npm workspaces.

### 2. Configure environment

Copy the example envs:

```bash
cp apps/api/.env.example apps/api/.env
cp apps/web/.env.example apps/web/.env
```

Update `JWT_ACCESS_SECRET` and `JWT_REFRESH_SECRET` in `apps/api/.env` — both should be long random strings. The defaults work for dev but please replace them.

### 3. (Optional) seed a demo user

```bash
npm --workspace @excalidrow/api run seed
```

Demo user: **demo@excalidrow.local / demo1234**.

### 4. Run dev servers

```bash
npm run dev
```

This boots:

| Service | URL                       |
|---------|---------------------------|
| API     | http://localhost:4000     |
| Web     | http://localhost:5173     |
| Health  | http://localhost:4000/healthz |

The Vite dev server proxies `/api`, `/uploads` and `/socket.io` to the API, so everything works on a single origin during development.

### 5. Production build

```bash
npm run build      # builds the web bundle
npm start          # runs the API; serve apps/web/dist behind a reverse proxy
```

## 🧠 AI integration roadmap

The placeholder controller lives at:

```
apps/api/src/controllers/ai.controller.js
```

It already exposes:

- `GET  /api/v1/ai/capabilities`
- `POST /api/v1/ai/generate { feature, prompt }`

To activate AI:

1. Pick a provider (Ollama, OpenAI, Anthropic, …).
2. In `aiGenerate`, call your model with a prompt that asks for **Excalidraw element JSON**.
3. Return `{ status: 'ok', elements: [...] }` and the front-end (`AISidebar.jsx`) will inject those elements straight onto the canvas.

## 🛡️ Security defaults

- `helmet()` with sane CORP for static asset serving.
- `cors` locked to `WEB_ORIGIN`.
- `hpp` against parameter pollution.
- `express-rate-limit` globally + stricter limits on `/auth/*` and `/uploads/*`.
- `argon2id` password hashing.
- JWT access + rotated refresh tokens with **reuse detection** (any reuse purges all sessions for that user).
- Mongoose `strictQuery`, soft-delete on boards.
- File uploads constrained by MIME allow-list + `MAX_UPLOAD_BYTES`.

## 📌 Why JavaScript?

This codebase is intentionally TypeScript-free. The shared package keeps types out of the picture and uses `zod` schemas at runtime as the single source of truth — both API and web import the same schema files.
